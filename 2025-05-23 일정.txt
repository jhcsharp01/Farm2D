인벤토리 기본 설계

인벤토리 클래스
: 여러 개의 슬롯을
  보유하고 있는 클래스
  슬롯의 제거 추가가
  쉽게 처리되기 위해
  리스트 사용

 인벤토리 생성 시
 전달받은 숫자만큼
 슬롯이 추가될 수 있도록
 진행합니다.

 Add(CollectType type)
 : 슬롯 타입과 추가 여부를 판단해 추가하는 기능
  -> 추가 여부에 대한 조사로 슬롯 등록
  -> 타입이 None인 경우에 조사 후 등록 


슬롯 클래스
: 아이템의 정보가 등록
될 클래스

필드 :
   - 수집물 유형
   - 개수
   - 최대 저장 개수


기본 생성자 값
타입 : None
개수 : 0
최대개수 : 10

bool Addable()
: 현재 아이템 개수가 max_count보다 작을 경우에 추가를 요청

Add(CollectType type)
:  추가 가능 여부에 대한 함수


1교시 작업
: 인벤토리 기본 시스템 구현

2교시
: UI 관련 작업

Canvas 설정
: Render Mode 
  -> Screen Space - Camera
  이 기능은 렌더 카메라에 대한
  설정을 추가해 캔버스를 렌더 카메라
  위치로 두는 것으로, 2D에서 화면
  비교하면서 배치하기 쉬워서
  자주 선택되는 모드입니다.

 Order in Layer를 플레이어보다
 앞이 되도록 설정

 Canvas
   -> Inventory(Empty)
        -> Inventory_BG(Image, 인벤토리 이미지)
	-> Slots(Empty, Grid Layout Group, 슬롯 묶음 관리)
	      -> Slot(슬롯 , 색 변경(테두리 표현))
	            -> Slot_Icon(이미지, 가로 세로 80)
                         -> Slot_Count(텍스트, 개수)


Slots Layout Group 설정

Padding 을 활용해  방향별 간격 설정 가능

Cell Size를 통해 그룹에 들어갈 오브젝트의 크기 설정 가능

Spacing을 통해 다음 오브젝트와의 간격 처리 가능

Start Axis를 통해 배치 순서를 결정해주세요(가로, 세로)

Child Alignment를 통해 그룹 안에 들어갈 오브젝트의
배치 순서를 설정할 수 있습니다.

Constraint를 이용해 요소 추가에 대한 확장 설정을
진행할 수 있습니다.


인벤토리에 대한 설정은 자유롭게 진행합니다

InventoryUI.cs 생성
:
 필드 :
      인벤토리 창(온오프 용도)
       플레이어(플레이어의 정보 연결)

 로직 :

 기능 :
     OnOFF() - public
     : UI 갱신 기능 호출(SlotRenual)
      인벤토리 오브젝트의 활성화 여부에 따라 키거나 끔.
     
      SlotRenewal()  - private
     : 슬롯 타입에 따라, 이미지 갱신 또는 비움 작업을 진행합니다.

:
 필드 :
         슬롯 아이콘

       슬롯 데이터를 받아와 해당 데이터가 가진 이미지와, 슬롯 아이템 개수를 통해
